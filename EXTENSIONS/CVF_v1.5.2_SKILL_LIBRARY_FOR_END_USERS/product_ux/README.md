## 📋 Canonical UI/UX Skills

CVF keeps UI/UX web guidance intentionally small. For new web builds,
redesigns, and non-coder handoff packets, prefer these first:

| Skill | Difficulty | Mô tả |
|-------|-----------|-------|
| [CVF Web UX Redesign System](cvf_web_ux_redesign_system.skill.md) | Advanced | Chuẩn hệ thống để biến yêu cầu web/redesign thành CVF-native design DNA, layout, component rules, và implementation guardrails |
| [Claude Design Handoff](claude_design_handoff.skill.md) | Advanced | Hấp thụ prototype Claude Design/HTML thành handoff sạch, không copy brand identity |
| [Accessibility Audit](accessibility_audit.skill.md) | Medium | Kiểm tra WCAG compliance khi cần QA chuyên sâu |
| [UX Heuristic Evaluation](ux_heuristic_evaluation.skill.md) | Medium | Đánh giá UX tổng quát theo Nielsen khi cần review sau build |

Legacy visual micro-skills such as style selection, palette generation,
typography pairing, interaction review, dark/light audit, and generic design
system generation have been moved under `legacy/` because their useful content
now belongs in root `DESIGN.md` and the canonical redesign system skill.

## 🎯 Cách sử dụng

1. Chọn skill phù hợp với nhu cầu
2. Đọc qua **Form Input** để biết cần prepare gì
3. Điền form trong CVF Web hoặc copy spec
4. Dùng **Checklist** để tự đánh giá
5. Xem **Tips & Examples** để có thêm insights
6. Paste spec vào AI để nhận phân tích chi tiết

## 📝 CVF Skill Format

Mỗi skill file có cấu trúc:

```markdown
# Skill Name
> Metadata (Domain, Difficulty, Version)

## 🎯 Mục đích
Khi nào nên dùng skill này

## 📋 Form Input
Các field cần điền

## ✅ Checklist Đánh giá
Danh sách tiêu chí để check

## ⚠️ Lỗi Thường Gặp
Các mistake phổ biến và cách fix

## 💡 Tips & Examples
Best practices và ví dụ cụ thể

## 📤 Expected Output từ AI
Bạn sẽ nhận được gì khi dùng với AI
```
