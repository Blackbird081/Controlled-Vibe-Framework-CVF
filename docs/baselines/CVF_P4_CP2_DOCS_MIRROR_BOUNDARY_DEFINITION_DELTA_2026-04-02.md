# CVF P4 CP2 Delta - Docs Mirror Boundary Definition

Memory class: SUMMARY_RECORD
Status: records the first concrete `PUBLIC_DOCS_MIRROR` boundary after `P4/CP1` opened the planning lane.

## Purpose

- turn the docs-mirror idea into a usable boundary map
- prevent accidental interpretation of `docs/` as a wholesale public-mirror candidate
- keep evidence-dense and governance-dense docs inside the private core

## Outcome

- future docs mirror:
  - curated subset only
- direct mirror candidates:
  - front-door root files and learning-oriented docs zones
- conditional zone:
  - selected explanatory `docs/reference/` files after per-file review
- private-core-only zone:
  - audits, reviews, baselines, logs, roadmaps, and dense internal records
