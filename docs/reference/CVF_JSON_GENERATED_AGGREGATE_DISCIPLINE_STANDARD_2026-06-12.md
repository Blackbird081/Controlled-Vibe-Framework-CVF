# CVF JSON Generated Aggregate Discipline Standard

Memory class: FULL_RECORD

Status: ACTIVE_STANDARD

Date: 2026-06-12

## Purpose

Prevent recurring agent errors in large governed JSON files by separating
reviewable source fragments from generated machine aggregates.

## Scope

This standard applies to governed JSON files that are:

- edited by agents across sessions;
- large enough that hand edits are hard to review;
- used by machine gates, registries, active session state, corpus indexes,
  routing contracts, or continuity front doors;
- likely to receive repeated append-only entries.

Examples include:

- `docs/corpus-intelligence/CVF_CORPUS_SCAN_REGISTRY.json`;
- `CVF_SESSION/ACTIVE_SESSION_STATE.json`;
- `CVF_SESSION/agent_workspace/ACTIVE_AGENT_WORKSPACE_STATE.json`;
- future registry, catalog, ledger, manifest, or state aggregate JSON files.

## Rule

A large governed JSON aggregate should not be hand-edited once a generated
source layout exists.

The expected pattern is:

1. a compact core/header JSON source file for stable global fields;
2. one small JSON source file per registry/state entry when entries are
   append-like or independently reviewable;
3. a deterministic generator that builds the aggregate;
4. a checker that fails when the aggregate drifts from sources;
5. focused tests for ordering, source-only fields, and drift detection.

## Source File Discipline

Source files should be ASCII-safe by default. Human-language or historical
Unicode content may be stored through JSON escapes when the source file itself
is agent-authored. The generated aggregate may preserve runtime-required
Unicode only when the aggregate is rebuilt by the generator.

Source-only metadata such as order fields must be stripped from the generated
aggregate unless the runtime contract explicitly requires them.

## Aggregate Discipline

The aggregate remains the machine-consumed file when existing checkers,
front doors, or runtime readers depend on that path.

Agents must not bypass the source layout by editing only the aggregate. A
changed aggregate without matching source update is a drift defect.

## Closure Requirement

Any future CVF work that creates or materially expands a large governed JSON
aggregate must record one of:

- `GENERATED_SOURCE_LAYOUT_ADDED`;
- `EXISTING_GENERATED_SOURCE_LAYOUT_REUSED`;
- `N/A_WITH_REASON`.

`N/A_WITH_REASON` is valid only when the JSON is small, single-purpose,
generated by another existing tool, or not expected to receive repeated agent
edits.

## Claim Boundary

This standard improves authoring reliability and reviewability. It does not
prove semantic correctness of JSON values, runtime behavior, provider behavior,
production readiness, public readiness, or autonomous mutation safety.
