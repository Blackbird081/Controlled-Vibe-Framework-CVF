## Taxonomy Principles

1. Public/private boundary, source verification, no-commit plus reviewer separation, and closure evidence are protected controls. A proposed lane may reduce ceremony only when all four controls remain intact.
2. Missing authority is not a reason to lighten ceremony. Missing authority moves the work to `HOLD_SOURCE_AUTHORITY`.
3. Runtime, provider, public-sync, and public claim work is not eligible for representative proof by default. It moves to `BLOCK_RUNTIME_PROVIDER_PUBLIC` or the relevant full governed lane.
4. Representative proof is proposal-only until a later accepted work order implements a routing standard.
5. Format cost should be reduced only where the cost is unrelated to safety, source fidelity, repository boundary, or closure evidence.

## Representative Proof Threshold Detail

Representative proof is eligible only when all of these are true:

- The tranche is docs-only.
- The worker is under `WORKER_MUST_NOT_COMMIT`.
- No runtime, checker, hook, source, test, template, public-sync, provider, live proof, merge, or push surface is changed.
- The intake class is advisory critique or private public-summary/vocabulary hypothesis.
- The work order names the representative sample before dispatch.
- The worker return includes command-backed changed-set evidence and no-commit evidence.

Representative proof is not eligible when any of these are true:

- The work changes or validates public-facing output.
- The work asserts provider readiness, runtime behavior, live governance behavior, API proof, MCP behavior, or production capability.
- The source authority is missing or ambiguous.
- The proposed lighter lane removes any protected control.
