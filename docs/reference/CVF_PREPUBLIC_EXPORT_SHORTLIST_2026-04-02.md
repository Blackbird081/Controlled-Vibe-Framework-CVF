# CVF Pre-Public Export Shortlist — 2026-04-02

Memory class: POINTER_RECORD
Status: canonical shortlist reference for the first bounded selective-export planning wave.

## Purpose

- define which `PUBLIC_EXPORT_CANDIDATE` extensions should be considered first for future packaging work
- stop the current candidate pool from being interpreted as one undifferentiated export backlog
- preserve the rule that shortlist membership is not publication approval

## Core Rule

Shortlist membership means:

- higher near-term planning priority
- bounded candidate for future packaging work

It does not mean:

- `READY_FOR_EXPORT`
- approved for public publication
- approved for package release

## First-Wave Shortlist

These are the preferred first bounded candidates for a later packaging-focused packet.

- `CVF_GUARD_CONTRACT`
- `CVF_v3.0_CORE_GIT_FOR_AI`
- `CVF_v1.7.3_RUNTIME_ADAPTER_HUB`

## Why These Three

- they already present clearer package identity than many other candidates
- they have narrower and more legible export stories than the large plane foundations
- they are easier to discuss as selective public modules without implying broad private-core release

## Deferred Candidate Groups

- `CVF_CONTROL_PLANE_FOUNDATION`
- `CVF_EXECUTION_PLANE_FOUNDATION`
- `CVF_GOVERNANCE_EXPANSION_FOUNDATION`
- `CVF_LEARNING_PLANE_FOUNDATION`

Reason:

- important long-term candidates, but first-wave packaging would be broader, riskier, and more support-heavy

### Deferred Group B — Medium Candidates Requiring More Boundary Work

- `CVF_AGENT_LEDGER`
- `CVF_MODEL_GATEWAY`
- `CVF_POLICY_ENGINE`

- `CVF_v1.7.1_SAFETY_RUNTIME`
- `CVF_v1.9_DETERMINISTIC_REPRODUCIBILITY`

Reason:

- still plausible candidates, but not the smallest first bounded wave

### Deferred Group C — Concept-Only Hold

- `CVF_PLANE_FACADES`

Reason:

- current registry truth remains `CONCEPT_ONLY`

## Guardrails

- do not treat this shortlist as approval for publication
- do not change `exportReadiness` without a separate packet
- do not use the shortlist to imply that the private monorepo is becoming public
- keep docs-mirror planning and export-packaging planning as separate lanes
- use `docs/reference/CVF_PREPUBLIC_SHORTLIST_PACKAGING_BOUNDARY_2026-04-02.md` when later export-planning work needs the current candidate-scoped packaging boundary
