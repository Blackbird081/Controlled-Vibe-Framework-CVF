## Purpose

Point to the canonical Role Switch Envelope standard and state the one rule that
matters most: routine inter-agent jurisdiction decisions do not go to the
non-coder operator.

## Scope

Applies to any agent acting as dispatch author, worker, reviewer, closer, or
session-sync steward in a governed CVF batch. This front door is a pointer to the
RSE standard; the standard holds the normative content.

## Read This

- Standard: `docs/reference/role_switch_envelope/CVF_ROLE_SWITCH_ENVELOPE_STANDARD.md`

The standard defines the canonical envelope fields an agent states when it
changes or accepts a role, plus the operator-question boundary rule, a compliant
worker-to-reviewer example, and a forbidden operator-question example.

## One-Line Rule

Capture findings in your allowed return surface and route out-of-scope promotion
to the reviewer or closer. Ask the operator only for operator-owned decisions:
scope, risk, live or provider use, public-sync, destructive action, secrets or
quota, roadmap or tranche selection, or explicit authorization.

## Boundary

RSE is documentation and reference only and is a local view of the ratified Agent
Handoff Contract. It adds no checker, helper, runtime, provider, public-sync, or
interception behavior. RSE-T1, RSE-T2, and RSE-T3 are separate tranches.
