## 4. Scope Boundaries

**In scope:**
- browser-local analytics data (W127 event stream)
- 6 noncoder lanes derived from the 6 W127 metrics
- threshold bands: healthy / watch / action_required / no_data
- operator readout in existing `/analytics` page (new tab)
- flag-aware recommendation logic (W122–W126 feature flags)
- closure packet naming next-tranche bias

**Out of scope:**
- server-side telemetry, ingestion, or database
- new noncoder capability lanes
- automatic feature-flag changes
- A/B experimentation framework
- routing corpus expansion beyond W126

## 6. Exit Criteria (from roadmap §6)

W128 closes only when:
- noncoder lanes are summarized in one coherent operator readout

- feature-flag posture is visible beside the metrics
- the readout can distinguish healthy / watch / action_required / no_data
- a closure packet identifies the most justified next optimization lane
