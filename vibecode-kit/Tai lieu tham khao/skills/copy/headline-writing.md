# Headline & Copy Writing Skill

> Craft compelling headlines and microcopy that converts visitors to users.

## Token Budget: ~250 tokens

## 1. HEADLINE FORMULAS

### The 4U Framework
Every headline should have at least 2 of these:
- **Useful** - Solves a problem
- **Urgent** - Creates time pressure
- **Unique** - Differentiates
- **Ultra-specific** - Concrete, not vague

### Proven Headline Structures

**Value Proposition:**
```
[Action] + [Outcome] + [Without Pain Point]

Examples:
- "Build landing pages in minutes without writing code"
- "Ship features 10x faster without sacrificing quality"
- "Scale your API without managing infrastructure"
```

**How-To:**
```
How to [Achieve Outcome] [Timeframe/Constraint]

Examples:
- "How to launch your SaaS in a weekend"
- "How top teams ship 3x faster with AI"
- "How to reduce churn by 40% in 30 days"
```

**Number/List:**
```
[Number] Ways to [Achieve Benefit]

Examples:
- "7 ways to double your conversion rate"
- "3 simple steps to automate your workflow"
- "The only 5 metrics that matter for growth"
```

**Question:**
```
[Pain Point Question]?

Examples:
- "Tired of slow deployments?"
- "Why do 80% of startups fail at scaling?"
- "What if you could code 10x faster?"
```

**Before/After:**
```
From [Pain State] to [Desired State]

Examples:
- "From idea to production in hours, not weeks"
- "From spreadsheets to real-time dashboards"
- "From manual testing to automated CI/CD"
```

### CTA Button Copy Rules
1. **Use first person** - "Start my free trial" > "Start your free trial"
2. **Add urgency** - "Get early access" > "Sign up"
3. **Reduce risk** - "Try free for 14 days" > "Start trial"
4. **Be specific** - "Download the template" > "Download now"

## 6. MICROCOPY

### Form Labels & Placeholders
```
Label: Work email
Placeholder: you@company.com
Helper: We'll never share your email

Label: Password
Placeholder: At least 8 characters
Helper: Use letters, numbers, and symbols
```

### Empty States
```

"No projects yet"
"Create your first project to get started. It only takes 30 seconds."

```

### Error Messages
```
Bad: "Error 500: Internal server error"
Good: "Something went wrong on our end. We're looking into it. Try again in a few minutes."

Bad: "Invalid input"
Good: "Please enter a valid email address (e.g., you@example.com)"

### Success Messages
```

## 7. PRODUCT-SPECIFIC COPY

### Landing Pages
- Hero: Value prop in 8 words or less
- Subhead: Expand with specifics (15-20 words)
- CTA: Action + benefit
- Features: Benefit-first, not feature-first

### SaaS Products
- Pricing: Anchor high, show savings
- Plans: Name for audience ("Startup", "Scale", "Enterprise")
- Feature lists: ‚úì checkmarks, specific limits

### Dashboards
- Navigation: Clear, action-oriented labels
- Tooltips: Helpful, not obvious
- Empty states: Guide to next action

### Documentation
- Titles: Task-oriented ("Getting started with X")
- Intro: What user will learn/achieve
- Code comments: Why, not what

## 8. COPY CHECKLIST

### Before Writing
- [ ] Who is the audience? (Role, pain points)
- [ ] What's the one thing they should do?
- [ ] What objections might they have?
- [ ] What proof can support claims?

### After Writing
- [ ] Can this be shorter? (Cut 20%)
- [ ] Is the benefit clear in first 5 words?
- [ ] Would the reader say "that's me"?
- [ ] Is there a clear next action?
- [ ] Does it sound human, not corporate?

### Words to Avoid
```
Overused: innovative, cutting-edge, best-in-class, seamless, robust
Vague: solution, platform, leverage, synergy, optimize
Corporate: utilize (use), facilitate (help), implement (build)
```

### Power Words
```
Verbs: Launch, Build, Ship, Create, Transform, Unlock, Boost
Results: Instantly, Automatically, Effortlessly, Finally
Proof: Proven, Guaranteed, Trusted, Verified
```
